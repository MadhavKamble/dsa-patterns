## 🗂️ Solution Format (Every Problem)

Each solution file contains:

1. **Problem Understanding** — what it asks, I/O, constraints, key observations
2. **Brute Force** — naive solution with complexity
3. **Better Approach** — improvement if exists
4. **Optimal Approach** — key insight, dry run, complexity proof
5. **Interview Explanation Script** — ready-to-speak explanation
6. **Clean C++ Code** — production-quality, C++17, LeetCode-ready
7. **Recall Line** — 10-second memory hook
8. **Short Revision Notes** — ≤10 bullet points
9. **Pattern Recognition Trigger** — when to apply this pattern
10. **Common Mistakes** — interview pitfalls
11. **What Interviewer Is Testing** — hidden evaluation criteria
12. **Alternative Approaches** — all valid solutions compared
13. **Related Problems** — LeetCode + Striver cross-references
14. **Pattern Category** — classified by technique
15. **Difficulty Analysis** — why it's rated as it is
16. **Progression Insight** — harder variants to prepare for
17. **30-Second Last-Minute Revision** — final cram block
